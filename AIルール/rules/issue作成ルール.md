Issueアウトライン表を元に、Issue本文を生成し、`docs/plan.md` に追記してください。

### ソース
- Issueアウトライン表は `docs/implementation_plan.md`です。
- `docs/` 配下の設計ドキュメントを必ず参照してください（特に `docs/モジュール構成と処理フロー.md` で対象モジュールの責務・依存関係を把握する）。
- 既存コードを参照してください。

### 対象
- Issue番号: #{{ISSUE_NUMBER}}

### 出力要件
1. **出力先は `docs/plan.md` のみ**（`docs/issues/` ディレクトリは使用しない）
2. `docs/plan.md` が既に存在する場合は、既存のIssueセクションの後に新しいIssueセクションを追記する
3. `docs/plan.md` が存在しない場合は、新規作成し、先頭に「# Issue Plan」と「## 共通テスト運用メモ」を追加する
4. ACは3〜7項目、`- [ ]` 形式で展開（ドキュメント更新がある場合はAC内に対象ファイル名と更新概要を必ず記載し、`docs/モジュール構成と処理フロー.md` に影響する際もここで明記する。別セクションは作らない）
5. 依存Issueは `#1, #2` のように列挙
6. 各Issueは `## Issue #{{ISSUE_NUMBER}}: {{タイトル}}` の見出しで開始する

### ISSUE_TEMPLATE

各Issueは以下の形式で `docs/plan.md` に追記する：

```markdown
## Issue #{{ISSUE_NUMBER}}: {{タイトル}}
### 背景 / 目的
{{1〜2文で理由}}

- 依存: {{依存Issueを列挙、なければ「-」}}
- ラベル: {{backend, frontend, infra 等}}

### スコープ / 作業項目
- {{作業項目1}}
- {{作業項目2}}
- {{作業項目3}}

### ゴール / 完了条件（Acceptance Criteria）
- [ ] {{AC1}}
- [ ] {{AC2}}
- [ ] {{AC3}}
- [ ] 実装後に関連テスト（例: `/Users/Shared/my-projects/.venv/bin/python -m pytest ...`）を実行し、結果を本IssueのAC更新で反映する

### テスト観点
- **ユニットテスト**: {{対象とテスト内容}}
- **統合テスト**: {{対象とテスト内容}}
- **手動テスト**: {{確認項目}}
- **検証方法**:
  - {{検証コマンドや手順}}

要確認事項:
- {{疑問点やTODO、なければ「なし」}}
- {{疑問点やTODO、なければ「なし」}}
```

### 追記時の注意事項
1. `docs/plan.md` の先頭に「# Issue Plan」が存在することを確認する
2. 「## 共通テスト運用メモ」セクションが存在することを確認する（存在しない場合は追加）
3. 既存のIssueセクションの最後に新しいIssueセクションを追加する
4. 各Issueセクション間は空行で区切る
